# AGENTS.md

Vue 3 + Quasar + FormKit input wrappers. Single package, two build targets:
**library** (published to npm) and **docs site** (deployed to Netlify).

## Commands

| Command | What it does |
|---|---|
| `npm run dev` / `npm run docs:dev` | Vite dev server for the **docs site** (`docs/`, port 5173). `dev` is **not** a library dev server — there isn't one. |
| `npm run build` | **Typecheck + library build** to `dist/`. Runs `vue-tsc && vite build`. |
| `npm run docs:build` | Build static docs to `docs-dist/`. No typecheck. |
| `npm run preview` | Vite preview of the docs build. |
| `npm run release` | `npm version patch && npm run build && npm publish`. Do not run by hand to publish — tag-driven CI handles releases. |

There is **no test, lint, or format script**. Do not invent `npm test` / `npm run lint`. If you need typechecking, `npm run build` covers it (it runs `vue-tsc` first and fails on errors).

## Layout

- `src/components/` — Vue 3 wrappers for each Quasar input (`Input.vue`, `Select.vue`, …). Library entry: `src/components/index.ts` exports `createQuasarPlugin` (a `FormKitPlugin` mapping `q-*` types → components) plus `registerErrorMessage` / `getErrorMessage` helpers.
- `src/utils/` — FormKit error-message helpers used by all wrappers.
- `docs/` — Vite docs site. `docs/pages/` = one file per input (route is derived from filename). `docs/router.ts` wires routes; `docs/main.ts` mounts Vue + Quasar + FormKit.
- `vite.config.ts` — library build, ES format only, externals listed (vue, @formkit/*, quasar, @quasar/extras).
- `vite.docs.config.ts` — docs build, root is `docs/`, alias `@` → `docs/`, output `docs-dist/`.

## Conventions / gotchas

- FormKit prop forwarding: attributes not consumed by FormKit are passed through to the underlying Quasar component via `context.attrs`. To map an extra FormKit prop to a Quasar prop, add it to the `props: [...]` array in the `node.define(...)` call inside `createQuasarPlugin` (see `q-input` → `inputType`, `q-option-group` → `optionType`).
- Error rendering: inputs that don't expose Quasar's `error`/`error-message` props (`q-color`, `q-date`, `q-time`, `q-slider`, `q-range`, `q-rating`, `q-btn-toggle`) display errors as a red caption via `getErrorMessage`. Others use Quasar's native props. Keep this split when adding new wrappers.
- Slot forwarding pattern (see `src/components/Input.vue:25`): iterate `useSlots()` and re-emit each slot with the Quasar component's scoped props — required for `prepend`, `append`, etc. to work.
- Library output: `dist/formkit-quasar.js` (ES), types at `dist/components/index.d.ts` (generated by `vite-plugin-dts`, `rollupTypes: true`). Don't hand-edit files in `dist/` — they are regenerated.
- `vue-tsc` will fail on unused imports/params (`noUnusedLocals`, `noUnusedParameters` are on). Don't leave dead code in `.vue` or `.ts` files.

## Adding a new input

1. Create `src/components/MyInput.vue` mirroring `Input.vue` (use `<script setup lang="ts">`, accept `context: FormKitFrameworkContext`, expose `value` via `computed` calling `node.input(val)`).
2. Register a `case "q-my-input":` branch in `createQuasarPlugin` in `src/components/index.ts`. List any extra prop names in `props: [...]` so FormKit doesn't strip them.
3. Add a doc page under `docs/pages/MyInput.vue` and a route in `docs/router.ts`.
4. `npm run build` must succeed (typecheck). `npm run docs:dev` to preview.

## Release flow

- CI publishes on push of a `v*` tag via `.github/workflows/publish.yml` (Node 22, `npm ci`, `npm run build`, `npm publish --access public`).
- Docs auto-deploy to Netlify on push to `main` via `.github/workflows/deploy-docs.yml` (`npm run docs:build` → `netlify-cli deploy --prod --dir=docs-dist`).
- Netlify config in `netlify.toml`: build command `npm run docs:build`, publish dir `docs-dist`, SPA fallback redirect.
- Don't commit secrets; CI reads `NETLIFY_AUTH_TOKEN` and `NETLIFY_SITE_ID` from repo secrets.

## Things explicitly not in this repo

No tests, no ESLint, no Prettier, no husky, no commitlint, no monorepo tooling. Don't add them without being asked.
